Three mental models

Same runtime, three ways to frame it for customers. Full write-up: proposals/mental-models.md.

🧠 Shared reality: in the runtime, tools/, skills/ and mcp.json live at the Function App level and are shared across its agents; each agent just filters which ones it uses. Every model must handle this.
1

Function App-centric

The Function App is the object. Agents + shared tools/skills live inside it, and sharing is visible.

  • ✅ Matches runtime exactly
  • ✅ Shared edits are safe (visible)
  • ⚠️ "Function App" is infra vocabulary
Open concept →
2

Agent-centric

The Agent is the front door. Pick/create a Function App on create; see only this agent's tools/skills.

  • ✅ Most intuitive front door
  • ✅ Focused per-agent view
  • ⚠️ Hides shared-edit impact
Open concept →
3

Project-centric

A Project holds agents + shared tools/triggers + many resources. Cleanest, but needs its own store.

  • ✅ Cleanest grouping of resources
  • ✅ Bounded blast radius
  • ⚠️ New persistence layer + cost
Open concept →

Recommendation (see proposal)

Start with Model 1's truth, wear Model 2's face (agent-first create), defuse the shared-edit footgun with "used by N agents" + impact warnings + copy-on-edit, and keep Model 3 (Projects) in reserve for multi-resource grouping.
