Why it's clean

  • One workspace spans multiple apps & resources
  • Shared tools/triggers have a clear project boundary
  • Add heterogeneous resources without leaving the model

What it costs

💾 A Project has no runtime counterpart, so it needs a persistence layer (e.g. Cosmos DB / Table / blob index) to track membership, apps, and shared resources — an extra always-on, billable component to keep in sync with the files.
